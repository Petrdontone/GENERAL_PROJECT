git init - Создать пустой репозиторий Git или вновь инициализировать существующий;
git add - Добавить отдельный файл File.txt или все файлы в папке . в область подготовленных файлов;
git status - Проверка статуса репозитория;
git clone - создание копии (удаленного) репозитория;
git commit - Создании коммита в репозитории, что бы добавить сообщение необходим флаг -m;
git log - Отображает список последних коммитов, флаг -p отображает подробные изменение;
git show - Просмотр заданного коммита;
git diff - Просмотр изменений до коммита, для просмотра подготовленных изменений флаг --staged;
git rm - Удаление отслеживаемых файло (dirname/somefile.js, dirname/*.html)
git mv - Переименование файлов;
git checkout File.txt - Восстановить файлы рабочего дерева, не подготовленные к коммиту;
git reset HEAD somefile.js - Восстановить подготовленный файл, если папку, то файл указывать не нужно;
git revert - Откат заданного коммита;
git branch - Просмотр списка веток;
git branch new_branch_name - Создать новую ветку;
git checkout -b new_branch_name - Создание ветки и автоматический переход;
git branch -a - Список удалённых веток;
git branch -d - Удаление ветки, для принудительного удаления ветки флаг -D с заглавной буквой;
git push origin --delete - Стереть удалённую ветку;
git merge - Слияние двух веток, --no-ff объединит заданную ветку с основной и произведёт коммит слияния;
git remote add <shortname> <url> - Добавление удалённого репозитория;
git remote show <origin> - Получение дополнительных сведений об удалённом репозитори;
git push <origin> <main> - Отправка изменений в удалённый репозиторий;
git push -u origin new_branch - Отправка новой ветки в удалённый репозиторий;
git push --delete origin existing_branch - Удаление удалённой ветки;
git pull - Получение изменений из удалённого репозитория;
